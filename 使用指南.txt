================================================================================
                    每日提醒助手（完整版）- 使用指南
================================================================================

本文档用于在本地配置时对照操作，详细说明请查看 README.md

================================================================================
                              准备工作
================================================================================

【需要的账号和密钥】

1. GitHub 账号
   - 注册地址：https://github.com

2. 和风天气 API（免费）
   - 注册地址：https://dev.qweather.com/
   - 获取内容：QWEATHER_KEY 和 QWEATHER_HOST

3. Server酱（免费）
   - 注册地址：https://sc3.ft07.com/
   - 微信扫码登录即可获取 SendKey
   - 说明：Server酱提供两种推送方式——微信公众号推送和 App 推送。
     实际体验下来，App 推送效果更好（公众号推送容易被其他消息刷掉）。
     本指南提供的网址是 App 版本对应的官网，可根据官网指南下载对应的移动端APP
4. 阿里百炼 API（完整版需要）
   - 注册地址：https://bailian.console.aliyun.com/
   - 开通服务后获取 DASHSCOPE_API_KEY

================================================================================
                           配置步骤（按顺序操作）
================================================================================

【第一步】Fork 项目
--------------------------------------------------------------------------------
1. 登录 GitHub
2. 打开本项目页面
3. 点击右上角「Fork」按钮
4. 点击「Create fork」

【第二步】设置 GitHub Secrets
--------------------------------------------------------------------------------
1. 进入你 Fork 的项目
2. 点击「Settings」
3. 左侧菜单：Secrets and variables → Actions
4. 点击「New repository secret」，添加以下 4 个：

   名称：QWEATHER_KEY
   值：你的和风天气API密钥

   名称：QWEATHER_HOST
   值：你的和风天气API地址（类似 p46yvyu2n3.re.qweatherapi.com）

   名称：SERVERCHAN_KEY
   值：你的Server酱SendKey

   名称：DASHSCOPE_API_KEY
   值：你的阿里百炼API密钥

【第三步】下载项目到本地
--------------------------------------------------------------------------------
1. 在你的项目页面，点击绿色「Code」按钮
2. 选择「Download ZIP」
3. 解压到任意文件夹

【第四步】创建 .env 文件
--------------------------------------------------------------------------------
1. 找到 .env.example 文件
2. 复制一份，重命名为 .env（去掉 .example）
3. 用记事本打开，填入你的密钥：

   QWEATHER_HOST=你的API地址
   QWEATHER_KEY=你的API密钥
   SERVERCHAN_KEY=你的SendKey
   DASHSCOPE_API_KEY=你的阿里百炼密钥

【第五步】安装 Python 和依赖
--------------------------------------------------------------------------------
1. 如果没有 Python，去 https://www.python.org/downloads/ 下载安装
   ※ 安装时务必勾选「Add Python to PATH」

2. 打开命令提示符（Win+R，输入 cmd，回车）

3. 进入项目文件夹：
   cd 你的项目路径
   例如：cd C:\Users\你的用户名\Downloads\每日提醒助手-完整版

4. 安装依赖：
   pip install -r requirements.txt

【第六步】运行配置网页
--------------------------------------------------------------------------------
1. 在命令提示符中运行：
   python web_config.py

2. 打开浏览器访问：http://localhost:5000

【第七步】在网页中配置
--------------------------------------------------------------------------------
◆ 学期设置：
  - 开学日期：格式如 2026-03-02
  - 总周数：一般 16-18 周
  - 城市代码：去 https://github.com/qwd/LocationList/blob/master/China-City-List-latest.csv 查询

◆ 课表管理：

  推荐方式：使用标准模板
    - 项目已包含「课表格式模板.xlsx」
    - 打开模板，删除示例课程，按格式填入自己的课表
    - 编号可随意填写（如[001]或[0]），不影响识别
    - 上传填写好的课表文件

  备用方式：
    - 上传教务系统导出的 Excel 课表
    - 手动添加课程

  提示：使用标准模板可获得最佳识别效果。
       后续版本将持续优化，支持更多格式的自动识别。
◆ 每日提醒：
  - 选择是否跳过周末推送
  - 添加提醒内容
  - 推送时间默认 23:30，如需修改见常见问题
◆ 提示词配置（完整版特有）：
  - 自定义 AI 激励内容风格
  - 自定义穿搭建议生成方式

◆ 测试：
  - 点击「测试推送」验证配置

【第八步】上传配置到 GitHub
--------------------------------------------------------------------------------
需要上传的文件：
- config.json（必须）
- llm_prompts.json（如果修改了提示词配置）

上传方法：
1. 在 GitHub 项目页面，点击要更新的文件
2. 点击右上角铅笔图标
3. 删除原有内容，粘贴本地文件的内容
4. 点击「Commit changes」保存

【第九步】测试自动运行
--------------------------------------------------------------------------------
1. 进入 GitHub 项目页面
2. 点击「Actions」标签
3. 左侧选择「Daily Reminder」
4. 点击右侧「Run workflow」→「Run workflow」
5. 检查微信是否收到推送

================================================================================
                              常见问题
================================================================================

Q: 没有收到推送？
A: 1) 检查 GitHub Secrets 是否正确
   2) 检查 Actions 运行是否成功
   3) 检查 Server酱 SendKey 是否有效

Q: 天气获取失败？
A: 1) 检查城市代码是否正确
   2) 检查和风天气 API 是否有效

Q: AI 内容生成失败？
A: 1) 检查阿里百炼 API 密钥是否正确
   2) 检查是否开通了相应服务

Q: 如何修改推送时间？
A: 推送时间由 GitHub Actions 控制，修改步骤：

   1) 进入 GitHub 项目页面
   2) 打开 .github/workflows/daily_reminder.yml 文件
   3) 点击右上角铅笔图标进入编辑模式
   4) 找到 cron: '30 15 * * *' 这一行
   5) 修改时间（格式：分 时 日 月 周，UTC时间）

   时间换算：
   - 北京时间 = UTC + 8
   - 例如：北京时间 21:00 = UTC 13:00
   - 例如：北京时间 23:30 = UTC 15:30

   建议：由于 GitHub Actions 用户量大，云端运行存在排队现象，
   建议将运行时间设置为目标时间提前 2-2.5 小时。
   例如希望 23:30 收到推送，可将 cron 设置为 '0 13 * * *'
   （UTC 13:00 = 北京时间 21:00），
   这样实际推送时间会在 23:00-23:30 之间。

   6) 修改完成后点击 Commit changes 保存

Q: 周末不想收到推送？
A: 在配置网页的「每日提醒」中选择「跳过周末」
